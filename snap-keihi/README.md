# パシャ経費

領収書の写真を撮る(か、ライブラリから選ぶ)だけで、AIが支払先・金額・カテゴリを
読み取って経費記録に残してくれる、仕事用の経費精算アプリです。溜まった領収書を
月末にCSVで書き出して経理に提出する、という流れを想定しています。

- ホーム画面には今月の経費合計・要精算(まだ払い戻しを受けていない立替分)・
  精算済み・法人カード払いの内訳、カテゴリ別の内訳(横棒グラフ)、日別の記録一覧
  が並び、月ごとに左右の矢印で切り替えられます。
- 右下の📷ボタンで撮影/画像選択 → サーバー側で Gemini API に画像を送って
  支払先・日付・金額・カテゴリ・メモ(摘要)を推測 → 内容を確認して保存、
  という流れです。読み取りに失敗しても手入力でそのまま保存できます。
- 「✏️ 手入力で追加」から、写真なしで直接入力することもできます。
- 各記録には「立替(個人払い)」か「法人カード」かの支払方法を設定できます。
  立替の記録は「未精算」バッジをタップするだけで「精算済み」に切り替えられます。
- 「よく使う項目」に定期券や毎月の通信費などを登録しておくと、毎月「追加する」
  をタップするだけで(金額はその場で調整可能)その月の記録として反映されます。
- 今月の合計カード内の「📤 この月をCSVで書き出す」から、表示中の月の記録を
  日付・支払先・金額・カテゴリ・支払方法・精算状況・メモ付きのCSVとして
  書き出せます。経理への提出や表計算ソフトでの管理に使えます。
- 1枚の領収書に交通費と会議費などカテゴリが混在している場合は、入力画面の
  「✂️ 別のカテゴリに分ける」から金額の一部を別カテゴリに分けて、2件以上の
  記録として保存できます。
- 記録はすべてブラウザの `localStorage` に保存されます(サーバーには保存されません)。
  端末やブラウザを変えると記録は引き継がれないので注意してください。
- 領収書画像はサムネイル(小さく圧縮した画像)だけを保存し、AI解析用に送る
  画像もその場で縮小してから送信しています。

## セットアップ

```bash
npm install
cp .env.example .env.local
# .env.local に GEMINI_API_KEY を設定 (https://aistudio.google.com/apikey で無料取得)
npm run dev
```

[http://localhost:3000](http://localhost:3000) を開くと使えます。
`GEMINI_API_KEY` が未設定でもアプリ自体は動きますが、写真からの自動読み取りは
できず、常に手入力での保存になります。

## 環境変数

| 変数名 | 必須 | 説明 |
| --- | --- | --- |
| `GEMINI_API_KEY` | - | Google Gemini APIキー。サーバーサイドのみで使用。未設定でも動くが自動読み取りは無効になる。 |
| `GEMINI_MODEL` | - | 読み取りに使うモデルID。未設定時は `gemini-flash-latest`(画像入力に対応したGoogle管理の最新Flashモデル)。 |

## Vercelへのデプロイ

1. このリポジトリ(このディレクトリ)をVercelにインポートする。
   - Root Directory に `snap-keihi` を指定する。
2. Vercelのプロジェクト設定 → Environment Variables に `GEMINI_API_KEY` を追加する。
3. デプロイ後に発行されるURLをスマホのホーム画面に追加しておくと、アプリのように
   ワンタップで開けて便利です。

## 使ってみたときのイメージ

1. 打ち合わせや出張のあと、📷ボタンをタップして領収書を撮る。
2. 数秒待つと支払先・金額・カテゴリが自動で入力される。支払方法(立替/法人カード)
   を選んで保存。
3. 月末になったら「📤 この月をCSVで書き出す」でCSVを書き出し、経理に提出する。
4. 払い戻しを受けたら、記録の「未精算」バッジをタップして「精算済み」にしておく。
